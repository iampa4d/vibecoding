# Lab 

### Lab 목표 : Agent 평가 + Human-in-the-loop + 운영 아키텍처

### Lab 시나리오

```text
AI Research Automation System

User
 │
Planner Agent
 │
 ├ Research Agent
 ├ Analysis Agent
 └ Writer Agent
 │
Tools
 ├ Browser MCP
 ├ Filesystem MCP
 └ GitHub MCP
 │
Evaluation
 │
Human Approval
 │
Monitoring
```

##### Step1 : Agent 평가 시스템 구축

```text
현재 AI Trend Automation 프로젝트에는 다음 Agent가 있다.

Research Agent
Analysis Agent
Writer Agent

Writer Agent가 생성한 보고서를 평가하는 Evaluation 시스템을 만들려고 한다.

평가 기준

1 정확도
2 일관성
3 신뢰도

LLM Judge 방식으로 평가하는 Python 코드를 작성해줘.

입력

market_analysis_report.md

출력

evaluation_result.md

파일

evaluation/evaluate_report.py
```


##### Step2 : Human-in-the-loop 시스템

```text
AI Agent가 생성한 보고서를 사람이 검토할 수 있도록 Human-in-the-loop 기능을 추가하려고 한다.

동작 방식

1 보고서 생성
2 사람이 검토
3 승인 또는 수정 요청

Python CLI 기반 인터페이스로 구현해줘.

파일

approval/review_system.py

동작

approve → 보고서 확정
reject → 수정 요청
```

##### Step3 : 운영 아키텍처 설계

```text
AI Trend Automation 시스템을 운영 환경에서 실행하기 위한 아키텍처를 설계해줘.

구성 요소

1 Agent System
2 API Server
3 Monitoring

포함 내용

- 전체 아키텍처 다이어그램
- 각 컴포넌트 역할
- 데이터 흐름

ASCII Diagram 형식으로 작성하고
system_architecture.md 파일로 만들어줘.
```

##### Step4 : Agent Monitoring 시스템

```text
AI Agent 시스템의 실행 상태를 모니터링하는 Monitoring 모듈을 만들어줘.

기능

1 Agent 실행 로그 기록
2 실행 시간 측정
3 오류 기록

출력

logs/agent_logs.txt

Python 코드로 작성해줘.

파일

monitoring/agent_monitor.py
```

##### Step5 : 비용 관리 시스템

```text
AI Agent 시스템에서 발생하는 LLM 호출 비용을 추적하는 Cost Tracker를 만들어줘.

기능

1 LLM 호출 횟수 기록
2 토큰 사용량 추정
3 총 비용 계산

출력

cost_report.md

파일

monitoring/cost_tracker.py
```
